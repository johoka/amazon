## 项目概述

### 核心功能
该Chrome插件自动化Amazon Seller Central的订单评价邀请流程：
- 自动筛选符合条件的订单
- 批量打开邀评标签页
- 自动确认邀请评价
- 持久化已邀请的订单记录

### 关键流程
```
order.js(解析订单)
  → background.js(管理标签页)
  → messagingReviews.js(自动确认)
  → injectedAjax.js(拦截API过滤)
```

### 🔴 **严重问题**

#### 1. **竞态条件导致的数据不一致** (order.js:45, 94-107)
**问题描述**:
```javascript
// 问题代码
while (!setMaxTabs) {
  console.info("等待获取同时邀请订单数");
  await sleep(1000);
}
```

**问题分析**:
- 使用轮询等待异步操作完成，低效且容易出问题
- `setMaxTabs`变量在`chrome.storage.local.get`的回调中修改，但回调可能失败或延迟
- 没有超时机制，可能陷入死循环
- 注释中明确指出bug：页面快速切换导致`invitedOrderIds.size`与`finalinvItedOrderIdSet.size`不一致

**影响**:
- 邀请状态不准确，导致订单重复邀请或遗漏
- 页面卡顿

**修复建议**:
```javascript
// 改进方案：使用Promise包装storage.get
function getMaxTabs() {
  return new Promise((resolve) => {
    chrome.storage.local.get("maxTabs", (result) => {
      resolve(result.maxTabs || 5);
    });
  });
}

// 使用
const maxTabs = await getMaxTabs();
```

#### 2. **内存泄漏 - 事件监听器未清理** (background.js:43-58)
**问题描述**:
```javascript
chrome.tabs.onUpdated.addListener(function (tabId, changeInfo, updatedTab) {
  if (tabId === tab.id && changeInfo.status === "complete") {
    chrome.tabs.sendMessage(tab.id, {
      startInvite: true,
      orderId: orderId,
      parentTabId: sender.tab.id,
    });
  }
});
```

**问题分析**:
- 每次打开新标签页都注册一个新的`onUpdated`监听器，但从不移除
- 随着时间推移，监听器堆积，造成严重的内存泄漏
- 旧的监听器继续响应所有标签页更新事件

**影响**:
- 长期使用插件导致Chrome进程占用内存不断增加
- 发送重复消息到已关闭的标签页
- 插件逐渐变慢

**修复建议**:
```javascript
// 创建一个监听器工厂，避免重复注册
const tabListeners = new Map();

function setupTabListener(tab, orderId, sender) {
  const listener = (tabId, changeInfo, updatedTab) => {
    if (tabId === tab.id && changeInfo.status === "complete") {
      chrome.tabs.sendMessage(tab.id, {
        startInvite: true,
        orderId: orderId,
        parentTabId: sender.tab.id,
      });
      // 关闭后移除监听器
      chrome.tabs.onUpdated.removeListener(listener);
      tabListeners.delete(tab.id);
    }
  };

  chrome.tabs.onUpdated.addListener(listener);
  tabListeners.set(tab.id, listener);
}
```

#### 6. **XHR/Fetch 拦截设计缺陷** (injectedAjax.js:24-54)
**问题描述**:
```javascript
window.fetch = async function (...args) {
  const response = await origFetch(...args).catch((error) => {
    window.postMessage({ type: "refreshOrder", data: {} }, "*");
  });
  // response 可能是 undefined
  console.log(response);  // 可能输出undefined

  if (isSearchOrderUrl) {
    response.clone().json()...  // response为undefined时崩溃！
  }
}
```

**问题分析**:
- 错误处理后仍继续执行，导致undefined错误
- 没有验证response有效性就直接clone()
- 将修改全局window.fetch，可能与其他脚本冲突
- 无法追踪哪些API调用被拦截

**修复建议**:
```javascript
const interceptedUrls = new Set([
  '/orders-api/search',
  '/orders-api/refund-status',
  '/orders-api/order/.*/feedback'
]);

window.fetch = async function (...args) {
  const url = args[0];
  const isIntercepted = interceptedUrls.has(new URL(url).pathname);

  if (!isIntercepted) {
    return origFetch(...args);
  }

  try {
    const response = await origFetch(...args);
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }
    return response;
  } catch (error) {
    console.error(`API请求失败: ${url}`, error);
    window.postMessage({
      type: "error",
      url: url,
      error: error.message
    }, "*");
    throw error; // 重新抛出让调用方处理
  }
};
```

#### 10. **存储容量有限，FIFO清理不合理** (background.js:102-104)
**问题代码**:

  RETENTION_DAYS: 90,
  CLEANUP_STRATEGY: 'oldest' // 可选: 'oldest', 'least_recent'
};

function cleanupStorage() {
  const cutoffTime = Date.now() - (90 * 24 * 60 * 60 * 1000);
  const filtered = orderIdArray.filter(item => item.timestamp > cutoffTime);

**修复建议**:
```javascript
async function fetchWithRetry(url, options = {}, maxRetries = 3) {
  let lastError;

  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      const response = await fetch(url, options);
      if (!response.ok && response.status >= 500) {
        throw new Error(`HTTP ${response.status}`);
      }
      return response;
    } catch (error) {
      lastError = error;
      const delay = Math.pow(2, attempt) * 1000; // 指数退避
      await sleep(delay);
    }
  }

  throw lastError;
}
```

### 建议2: 实现事件驱动架构 (优先级: 高)
**当前问题**: 轮询低效，消息处理分散
**解决方案**: 使用EventEmitter模式

// events.js
class EventBus {
  constructor() {
    this.listeners = {};
  }

  on(event, handler) {
    if (!this.listeners[event]) {
      this.listeners[event] = [];
    }
    this.listeners[event].push(handler);
  }

  emit(event, data) {
    if (this.listeners[event]) {
      this.listeners[event].forEach(h => h(data));
    }
  }
}

// 使用

// retry.js
async function retry(fn, options = {}) {
  const { maxAttempts = 3, delay = 1000 } = options;

  for (let i = 0; i < maxAttempts; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxAttempts - 1) throw error;
      await sleep(delay * Math.pow(2, i));
    }
  }
}

// 使用
const maxTabs = await retry(() => getMaxTabsAsync(), { maxAttempts: 5 });

        if (el) return resolve(el);
