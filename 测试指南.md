# 修改后测试指南

## 🚀 快速开始

### 步骤1：加载修改后的扩展
1. 打开 Chrome
2. 访问 `chrome://extensions/`
3. 启用"开发者模式"（右上角）
4. 点击"加载已解压的扩展程序"
5. 选择 `amazon` 文件夹
6. 确认加载成功（应该看到"邀评工具"扩展）

### 步骤2：打开DevTools
1. 进入 https://sellercentral.amazon.com/orders-v3/
2. 按 `F12` 打开开发者工具
3. 选择"控制台"标签页
4. 保持DevTools开启进行测试

## 🔧 常见问题排查

### 问题1: 标签页无法打开
**症状**: 点击按钮后没有新标签页打开

**排查步骤**:
1. 检查DevTools控制台，查找错误信息
2. 检查Chrome权限设置
3. 尝试手动打开邀评URL
4. 查看background.js的错误日志

**修复方案**:
```javascript
// 在DevTools中手动测试
chrome.tabs.create({
  url: 'https://sellercentral.amazon.com/messaging/reviews?orderId=123-456&marketplaceId=ATVPDKIKX0DER',
  active: false
});
```

### 问题2: 内存持续增长
**症状**: 运行一段时间后Chrome占用内存过高

**原因分析**:
- 修复前：事件监听器堆积
- 修复后：应该已解决

**验证修复**:
```javascript
// 查看监听器数量
chrome.tabs.onUpdated.getListeners().length
// 应该 < 5（合理范围）
```

### 问题3: 邀请失败
**症状**: 邀请按钮点击后没有反应

**排查步骤**:
1. 检查选择器是否匹配（可能Amazon更新了UI）
2. 查看waitForElement的超时消息
3. 检查DOM结构是否改变

**修复方案**:
```javascript
// 在DevTools中测试选择器
document.querySelector('#ayb-reviews > div > kat-button[label="是"]')
document.querySelector('kat-button[label*="是"]')
// 如果都返回null，可能需要更新选择器
```
